title: Runs
description: Execute scenarios and retrieve results.
---

## POST /api/runs

Run a scenario. This is synchronous — it returns when the run completes.

```bash
curl -X POST http://localhost:8000/api/runs \
  -H "Authorization: Bearer $BENCH_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{ "scenario_id": "your-scenario-uuid" }'
```

**Request body**

| Field | Required | Description |
|---|---|---|
| `scenario_id` | Yes | UUID of the scenario to run |
| `agent_id` | No | Override the scenario's default agent |

**Response 201** — the completed run object including `status`, `passed_turns`, `total_turns`, and per-turn results.

---

## POST /api/runs/suite

Start all scenarios in a suite in parallel. Returns immediately with an array of run objects in `pending` status — execution happens in the background.

```bash
curl -X POST http://localhost:8000/api/runs/suite \
  -H "Authorization: Bearer $BENCH_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{ "suite_id": "your-suite-uuid" }'
```

**Response 201** — array of run objects (one per scenario), all with `status: "pending"`. Poll `GET /api/runs/{id}` for each run to get the final result, or use the WebSocket events `suite:scenario:done` and `suite:done` for real-time updates.

## GET /api/runs

List runs with optional filters.

```bash
curl "http://localhost:8000/api/runs?status=failed&limit=50" \
  -H "Authorization: Bearer $BENCH_TOKEN"
```

**Query params:** `status`, `scenario_id`, `suite_id`, `agent_id`, `workspace_id`, `limit` (default 50)

---

## GET /api/runs/{id}

Get a single run with full turn detail.

```bash
curl http://localhost:8000/api/runs/$RUN_ID \
  -H "Authorization: Bearer $BENCH_TOKEN"
```

---

## DELETE /api/runs/{id}

Delete a run.

```bash
curl -X DELETE http://localhost:8000/api/runs/$RUN_ID \
  -H "Authorization: Bearer $BENCH_TOKEN"
```

**Response 204** (no body)

---

## GET /api/failures

Returns failed and errored runs — the failures inbox. Includes a summary of the first failing turn (user input, judge reasoning, error message).

```bash
curl "http://localhost:8000/api/failures?limit=50" \
  -H "Authorization: Bearer $BENCH_TOKEN"
```

**Query params:** `suite_id`, `scenario_id`, `agent_id`, `limit` (default 50)

Each item in the response includes:

| Field | Description |
|---|---|
| `run_id` | ID of the failed run |
| `scenario_id` / `scenario_name` | Scenario that failed |
| `status` | `failed` or `error` |
| `first_failed_turn_index` | Index of the first failing turn |
| `first_failed_user_input` | User message that triggered the failure |
| `first_failed_reasoning` | Judge reasoning for the failed expectation |
| `first_failed_error` | Execution error message if applicable |
| `duration_ms` | Total run duration |
