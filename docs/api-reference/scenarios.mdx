---
title: Scenarios
description: Create and manage test scenarios.
---

## POST /api/scenarios

Create a scenario.

```bash
curl -X POST http://localhost:8000/api/scenarios \
  -H "Authorization: Bearer $BENCH_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Booking flow",
    "agent_id": "your-agent-uuid",
    "turns": [
      {
        "user_input": "I want to book a flight to NYC",
        "expectations": [
          { "type": "function_call", "function_name": "search_flights" },
          { "type": "message", "role": "assistant", "intent": "confirms it will search for flights" }
        ]
      }
    ]
  }'
```

---

## GET /api/scenarios

List all scenarios.

**Query params:** `tag` (filter by tag), `workspace_id`

---

## GET /api/scenarios/{id}

Get a scenario with full turn and expectation detail.

---

## PUT /api/scenarios/{id}

Update a scenario. Replaces the full scenario (turns + fields). Creates a new version automatically.

---

## DELETE /api/scenarios/{id}

Delete a scenario and all its runs.

---

## GET /api/scenarios/{id}/versions

List all versions of a scenario.

---

## POST /api/scenarios/{id}/versions/{version}/restore

Restore a previous version of a scenario.

---

## GET /api/scenarios/{id}/export

Export a scenario as a portable JSON payload (same shape as `POST /api/scenarios`). Useful for migrating scenarios between environments.

---

## POST /api/scenarios/import

Import a previously exported scenario. Accepts the same body as `POST /api/scenarios`.
