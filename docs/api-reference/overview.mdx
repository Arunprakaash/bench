**Base URL:** `http://localhost:8000`

**Interactive docs (Swagger):** `http://localhost:8000/api/docs`

---

## Authentication

All endpoints (except register and login) require a bearer token:

```
Authorization: Bearer <token>
```

Two token types are accepted:

| Type | How to get | Expiry |
|---|---|---|
| JWT | `/api/auth/login` or `/api/auth/register` | 7 days |
| API token | `/api/auth/api-token` (POST) | Never (until revoked) |

API tokens are prefixed with `ab_` and are recommended for CI/CD use.

---

## Response format

All responses are JSON. Errors follow this format:

```json
{
  "detail": "Error message here"
}
```

Standard HTTP status codes are used: `200`, `201`, `204`, `400`, `401`, `403`, `404`, `422`, `500`.

---

## Route groups

| Group | Base path | Description |
|---|---|---|
| Auth | `/api/auth` | Register, login, profile, API tokens |
| Agents | `/api/agents` | Create and manage agents |
| Scenarios | `/api/scenarios` | Define multi-turn test scenarios |
| Suites | `/api/suites` | Group scenarios for batch runs |
| Runs | `/api/runs` | Execute scenarios and retrieve results |
| Failures | `/api/failures` | Failures inbox — failed and errored runs |
| Automation | `/api/automation` | Scheduled runs and regression alerts |
